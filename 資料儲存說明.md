# 資料儲存說明

## 儲存方式

系統支援兩種資料儲存方式：

### 1. 本地儲存 (localStorage) - 預設

- **優點**：
  - 設定簡單，無需額外配置
  - 資料儲存在瀏覽器本地
  - 關閉瀏覽器後資料仍保留

- **缺點**：
  - 僅限單一瀏覽器/裝置
  - 清除瀏覽器資料會刪除
  - 不同裝置無法同步

### 2. GitHub Gist 儲存

- **優點**：
  - 資料儲存在 GitHub，可跨裝置同步
  - 資料不會因清除瀏覽器而遺失
  - 可在 GitHub 上查看和備份資料

- **缺點**：
  - 需要建立 GitHub Personal Access Token
  - 需要網路連線

## 如何設定 GitHub Gist 儲存

### 步驟 1：建立 GitHub Personal Access Token

1. 前往 GitHub：https://github.com/settings/tokens
2. 點擊「Generate new token」> 「Generate new token (classic)」
3. 設定 Token 名稱（例如：TSA System Storage）
4. 選擇過期時間（建議選擇較長時間）
5. 勾選權限：**`gist`**（只需要這個權限）
6. 點擊「Generate token」
7. **複製 Token**（只會顯示一次，請妥善保存）

### 步驟 2：在系統中設定

1. 進入「系統設定」頁面
2. 點擊「資料儲存設定」
3. 選擇「GitHub Gist 儲存」
4. 貼上剛才複製的 Token
5. 點擊「測試連線」確認成功
6. 點擊「儲存設定」

### 步驟 3：完成

設定完成後，所有資料會自動儲存到 GitHub Gist：
- 帳號資料
- 角色權限
- 操作日誌

## 注意事項

⚠️ **重要提醒**：
- Token 請妥善保管，不要分享給他人
- 如果 Token 洩露，請立即到 GitHub 撤銷並重新建立
- 切換儲存方式不會自動遷移資料，請手動備份
- 首次使用 Gist 儲存時，系統會自動建立新的 Gist

## 資料位置

使用 Gist 儲存後，可以在以下位置查看資料：
- GitHub Gist：https://gist.github.com
- 資料會以私有 Gist 形式儲存（只有您可以看到）

## 故障排除

### 連線測試失敗
- 檢查 Token 是否正確
- 確認 Token 有 `gist` 權限
- 確認網路連線正常

### 儲存失敗
- 系統會自動降級到 localStorage
- 檢查瀏覽器控制台是否有錯誤訊息
- 確認 Token 未過期
