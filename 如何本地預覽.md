# 如何本地預覽系統

本指南說明如何在本地瀏覽和測試 TSA 會館修繕系統。

## 🎯 目標

建立一個可以在本地瀏覽器查看和測試的系統版本。

---

## 方法一：開發模式（推薦）

這是最簡單的方式，適合開發和測試。

### 前置需求

- Node.js 18+ 已安裝
- npm 已安裝

### 步驟

1. **打開終端/命令提示字元**

2. **進入專案目錄**
   ```bash
   cd /path/to/tsa-repair-system
   ```

3. **安裝依賴**（如果尚未安裝）
   ```bash
   npm install
   ```

4. **啟動開發伺服器**

   **使用啟動腳本：**
   - macOS/Linux: `./start.sh`
   - Windows (CMD): `start.bat`
   - Windows (PowerShell): `.\start.ps1`

   **或直接使用 npm：**
   ```bash
   npm run dev
   ```

5. **打開瀏覽器**

   開發伺服器啟動後，您會看到類似以下的訊息：
   ```
   VITE v6.x.x  ready in xxx ms

   ➜  Local:   http://localhost:3000/
   ➜  Network: http://192.168.x.x:3000/
   ```

   在瀏覽器中打開 `http://localhost:3000` 即可。

### 優點

- ✅ 即時熱重載（修改程式碼後自動更新）
- ✅ 快速啟動
- ✅ 適合開發和測試

## 方法二：構建生產版本

如果您想要一個可以在任何地方運行的靜態版本。

### 步驟

1. **構建專案**
   ```bash
   npm run build
   ```

   這會創建一個 `dist` 目錄，包含所有靜態文件。

2. **預覽構建結果**

   **方法 A：使用 Vite 預覽**
   ```bash
   npm run preview
   ```
   然後在瀏覽器打開 `http://localhost:4173`

   **方法 B：使用本地伺服器**

   **Python 3：**
   ```bash
   cd dist
   python3 -m http.server 8000
   ```
   然後在瀏覽器打開 `http://localhost:8000`

   **Node.js (http-server)：**
   ```bash
   npm install -g http-server
   cd dist
   http-server -p 8000
   ```

   **PHP：**
   ```bash
   cd dist
   php -S localhost:8000
   ```

### 優點

- ✅ 生產環境版本
- ✅ 可以部署到任何靜態主機
- ✅ 性能優化

## 常見問題

### Q: 端口 3000 已被占用

**解決方案：**
1. 修改 `vite.config.ts` 中的端口
2. 或關閉占用端口的程序

### Q: 頁面顯示空白

**檢查：**
1. 瀏覽器控制台是否有錯誤（按 F12）
2. 確認開發伺服器正在運行
3. 確認訪問的是正確的 URL

### Q: 無法安裝依賴

**解決方案：**
```bash
# 清除快取並重新安裝
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```
